

ЛОГИКА СКРИПТА
СТАДИЯ 0
- Игрок отправляет сообщение

СТАДИЯ 1 _stage1_request_descriptions
Модель получает
- Назначенные промты с правой панели в соответствующем порядке, кроме системного промта.
- Сообщение от игрока: она должна воспринимать ее как запланированное игроком действие, которое он попытается совершить.
- Системный промт, созданный для этой стадии. (последний в списке, чтобы модель его последним прочитала и запомнила) (Тут он упоминается потому что идет после сообщения от игрока)
- Скрипт присылает список всех доступных объектов (предметы, персонажи, локации) (id и имена).
- Модель должна решить на основании имеющейся информации определить, какую сцену разыграть.
- Поняв, какая будет сцена, модель должна решить, какие промты объектов запросить у скрипта.
- Запрашивает промты объектов у скрипта командой Tools
- Получив всю информацию, начинает обдумывать сцену
- Приняв решение, она отправляет Список объектов, с помощью Tools

Если произошла ошибка - попытаться определить результат парсингом сообщения.
Если за отведенные попытки, решение не будет принят, и скрипт возвращался со стадии 2 два раза, принять последний Список объектов и отправляться на стадию 3


-> Список объектов
-> Список имен всех объектов


СТАДИЯ 2 _stage1_validate_scene
Скрипт отправляет новое сообщение модели. Модель получает:
- Назначенные промты с правой панели в соответствующем порядке.
- Список объектов
- Она проверяет все ли персонажи тут к месту (поскольку модель их по именам собирала, не зная деталей), соответственно системному промту.
- Если все впорядке, модель отправляет команду одобрения Tools
- Если не впорядке, модель отправляет команду неодобрения Tools 

Если произошла ошибка - попытаться определить результат парсингом сообщения.
Если за отведенные попытки, решение принято не будет, будет взят последний список персонажей и совершен переход на стадию 3.
Скрипт получает одобрение - переходит на стадию 3
Скрипт получает неодобрение - переходит на стадию 2. Всего до трех попыток. 
-> Список объектов

СТАДИЯ 3
Скрипт отправляет новое сообщение модели. Модель получает:
- Назначенные промты с правой панели в соответствующем порядке.
- Модель получает сообщение от игрока
- Модель проверяет не врет ли игрок, соответственно системному промту.
- Если игрок врет, изменяет его сообщение на такое, какое будет соответствовать информации.
Например, если игрок "открыл" запертую дверь, в итоге, он только "попытается ее открыть"
- Отправляет отредактированное сообщение игрока через Tools скрипту

-> Список объектов
-> Список имен всех объектов
-> Сообщение от игрока (или отредактированное)
Если произошла ошибка - попытаться определить результат парсингом сообщения.
Если за отведенные попытки, решение принято не будет, будет взят последний сценарий и совершен переход на стадию 4.
Скрипт получает одобрение - переходит на стадию 4
Скрипт получает информацию - заменяет сообщение игрока, и переходит на стадию 4


СТАДИЯ 4 _stage1_player_action
Скрипт отправляет новое сообщение модели. Модель получает:
- Назначенные промты с правой панели в соответствующем порядке.
- Сценарий
- Сообщение от игрока
- Набор сгенерированных чисел d20 
- Модель описывает что игрок сделал в текущих условиях. 
- Если игрок совершает действие, модель берет число из списка по порядку, и на основаннии правил бросания кубиков определяет успешность исхода.
- Если правила бросков кубиков не предоставлены,  симулирует бросок и определяет успешность исхода.
- Описывает что происходит с игроком на основании полученных данных
- Описание отправит через Tools скрипт.

Если произошла ошибка - попытаться определить результат парсингом сообщения.
Если за отведенные попытки, решение принято не будет, будет взят последний сценарий и совершен переход на стадию 5
-> Список объектов
-> Список имен всех объектов
-> Результат действий игрока
Переход на 5-ый этап

СТАДИЯ 5 _stage1_random_event
Скрипт отправляет новое сообщение модели. Модель получает:
- Назначенные промты с правой панели в соответствующем порядке.
- Список имен всех объектов
- Набор сгенерированных чисел d100
- Если правила бросков кубиков не предоставлены, симулирует бросок и определяет успешность исхода.
- Берет число d100, определяя вероятность совершения в события. В системном промте указано, до какого числа бросок будет успешным.
- Если результат неуспешный - переход на стадию 7
- Отправляет случиться событие или нет скрипту через Tools

Если произошла ошибка - попытаться определить результат парсингом сообщения.
Если за отведенные попытки, решение принято не будет, событие не произошло. Переход на стадию 7
-> Список объектов
-> Список имен всех объектов
-> Результат действий игрока
-> Событие (опционально)
Если есть событие - переход на стадию 5.1
Если нет события - Переход на стадию 7


СТАДИЯ 5.1 (если событие случилось)
Скрипт отправляет новое сообщение модели. Модель получает:
- Назначенные промты с правой панели в соответствующем порядке.
- Событие 
- Список объектов
- Список имен всех объектов
- Модель определяет, хватет ли ей Список объектов для описания события. И если нет, подбирает нужных в списке имен.
- Запрашивает у скрипта их описание новых объектов. Отправляет Новые объекты в скрипт через Tools
Если произошла ошибка - попытаться определить результат парсингом сообщения.
Если за отведенные попытки, решение принято не будет, событие не произошло. Переход на стадию 7

-> Cписок объектов + Новые объекты (опционально)
-> Список имен всех объектов
-> Результат действий игрока
-> Событие (опционально)
Если событие случилось - его надо прогнать через СТАДИЯ 2 для проверки уместности новых объектов. После чего на стадию 5.2

Если произошла ошибка - попытаться определить результат парсингом сообщения.
Если за отведенные попытки, решение принято не будет, событие не произошло. Переход на стадию 7
-> Cписок объектов + Новые объекты (опционально)
-> Список имен всех объектов
-> Результат действий игрока
-> Готовое событие (опционально)
