### Шаг 1. Прочитай инструкцию

### Шаг 2. Определи тип инструкции

**Тип A — Простая замена (без накопления)**
- Инструкция запрашивает одно значение (например, «цвет», «состояние», «местоположение», «одет ли персонаж и во что», «чист ли персонаж», «запах»).
- **Действие:**
  - Если в финальной истории **явно описано** это значение → извлеки его и используй как новое.
  - Если явного описания нет, но в поле «Память» для этого объекта уже есть строка с **этой же инструкцией** → **оставь значение из памяти без изменений** (выведи строку с тем же значением).
  - Если нет ни явного описания, ни памяти → **выведи разумное значение по умолчанию**, которое соответствует контексту (время суток, локация, типичное поведение).
    * Пример для персонажа утром на кухне: «одет в повседневную одежду (футболка, шорты/джинсы), чист, запах свежий, смешанный с ароматом еды».
    * Пример для локации: «список персонажей в доме» — извлеки из истории всех, кто упомянут как находящиеся внутри.

**Тип B — Накопление событий/действий с временными метками**
Признаки (наличие хотя бы одного):
- В инструкции есть требование указывать время (например, «укажи время», «с указанием времени», «[День, Месяц]»).
- В инструкции есть условие хранения (например, «за 24 часа», «за последний час»).
- Инструкция описывает последовательность действий или событий.

**Для типа B ты должен:**
1. Извлечь из финальной истории **те действия/события, которые соответствуют смыслу инструкции** (действие должно подпадать под описание, данное в инструкции).
2. К каждому подходящему действию приписать временную метку (ЧЧ:ММ), извлечённую из начала финальной истории.
3. Если в инструкции указан **конкретный формат временной метки** (например, `[День, ДД Месяц]` и `(ЧЧ:ММ)`), следуй ему. Если формат не указан — используй `(ЧЧ:ММ)`.
4. Найди в памяти текущую строку для этой инструкции. Сравни дату (день и месяц) относительно даты из финальной истории. Если прошло больше времени, чем указано в условии хранения — замени строку полностью. Если дата в пределах допустимого — добавь новые действия в конец через разделитель `;`.
5. **НЕ ДОБАВЛЯЙ действия, которые не соответствуют инструкции.** Если сомневаешься — пропусти.
6. Если новых действий нет, но есть предыдущие в пределах срока хранения — сохрани их без изменений.

### Шаг 3. Сформируй выходную строку
Формат: `object_id: {текст инструкции} -> {значение}`

- Для типа A: значение — строка. Никаких «нет информации».
- Для типа B: значение — строка, начинающаяся с даты (если требует инструкция), затем список `(ЧЧ:ММ) действие`. Разделитель — `; `.
